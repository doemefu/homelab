# M6 — App Infrastructure Design

**Date:** 2026-03-27
**Status:** Approved
**Scope:** Deploy PostgreSQL 17, InfluxDB 2, Mosquitto 2 in the `apps` namespace as shared infrastructure prerequisites for application services.

---

## Architecture

One new playbook `infra/playbooks/50_apps_infra.yml` runs on `k3s_server` with `run_once: true` and `delegate_to: localhost` — identical pattern to `40_platform.yml`.

Deployment order per service:
1. Namespace `apps` (idempotent, created once)
2. Kubernetes Secret from SOPS vars
3. ConfigMaps (init scripts, config files, ACL)
4. PersistentVolumeClaim (Longhorn)
5. Deployment
6. Service
7. Cloudflare Tunnel ingress update (Mosquitto only)

All raw manifests applied via the `kubernetes.core.k8s` Ansible module — no Helm charts.

## Services

### PostgreSQL 17

| Property | Value |
|----------|-------|
| Image | `postgres:17-alpine` |
| Namespace | `apps` |
| Port | 5432 |
| Storage | Longhorn PVC, 5Gi, `/var/lib/postgresql/data` |
| Requests | 100Mi RAM, 50m CPU |
| Limits | 512Mi RAM, 500m CPU |

**Databases:** `authdb`, `devicedb`, `datadb` — created via a ConfigMap init script mounted at `/docker-entrypoint-initdb.d/init.sql`. The postgres image runs all `.sql` files in that directory on first start.

**Secret** (`postgres-secret`):
- `POSTGRES_USER` ← `postgres_user`
- `POSTGRES_PASSWORD` ← `postgres_password`

**Service:** ClusterIP, port 5432. Internal only.

### InfluxDB 2

| Property | Value |
|----------|-------|
| Image | `influxdb:2-alpine` |
| Namespace | `apps` |
| Port | 8086 |
| Storage | Longhorn PVC, 10Gi, `/var/lib/influxdb2` |
| Requests | 200Mi RAM, 100m CPU |
| Limits | 1Gi RAM, 1000m CPU |

**Bootstrap:** InfluxDB 2 auto-initialises on first start when `DOCKER_INFLUXDB_INIT_MODE=setup` is set. Org `terrarium` and bucket `iot-bucket` are created automatically.

**Secret** (`influxdb-secret`):
- `DOCKER_INFLUXDB_INIT_ADMIN_TOKEN` ← `influx_admin_token`
- `DOCKER_INFLUXDB_INIT_PASSWORD` ← `influx_admin_password`
- `DOCKER_INFLUXDB_INIT_ORG` ← `influx_org`
- `DOCKER_INFLUXDB_INIT_BUCKET` ← `influx_bucket`
- `DOCKER_INFLUXDB_INIT_MODE` = `setup` (static, in manifest)
- `DOCKER_INFLUXDB_INIT_USERNAME` = `admin` (static)

**Service:** ClusterIP, port 8086. Internal only.

### Mosquitto 2

| Property | Value |
|----------|-------|
| Image | `eclipse-mosquitto:2` |
| Namespace | `apps` |
| Ports | 1883 (MQTT internal), 9001 (WebSockets → Cloudflare Tunnel) |
| Storage | Longhorn PVC, 1Gi, `/mosquitto/data` (message persistence) |
| Requests | 20Mi RAM, 10m CPU |
| Limits | 128Mi RAM, 100m CPU |

**External access:** `mqtt.furchert.ch` → `ws://mosquitto.apps.svc.cluster.local:9001` via Cloudflare Tunnel. ESP32 devices connect via `wss://mqtt.furchert.ch` using a WebSocket-capable MQTT library.

**Auth:** Password file at `/mosquitto/config/passwd`, generated by an init container.

**Init container** (`eclipse-mosquitto:2`):
- Reads plaintext passwords from secret env vars
- Runs `mosquitto_passwd -b /mosquitto/config/passwd <user> <password>` for each of the 3 users
- Writes to a shared `emptyDir` volume mounted at `/mosquitto/config`
- Main container mounts the same volume — passwd file is ready before Mosquitto starts

**ConfigMap `mosquitto-config`** (`mosquitto.conf`):
```
listener 1883
listener 9001
protocol websockets
persistence true
persistence_location /mosquitto/data/
password_file /mosquitto/config/passwd
acl_file /mosquitto/config/acl.conf
allow_anonymous false
```

**ConfigMap `mosquitto-acl`** (`acl.conf`):
```
user terra1
topic readwrite terra1/#
topic read terraGeneral/#

user terra2
topic readwrite terra2/#
topic read terraGeneral/#

user backend
topic readwrite #
```

**Secret** (`mosquitto-secret`):
- `MOSQUITTO_BACKEND_PASSWORD` ← `mosquitto_backend_password`
- `MOSQUITTO_TERRA1_PASSWORD` ← `mosquitto_terra1_password`
- `MOSQUITTO_TERRA2_PASSWORD` ← `mosquitto_terra2_password`

**Service:** ClusterIP, ports 1883 + 9001.

|----------|-------|

| `mosquitto_backend_password` | MQTT password for `backend` user |
| `mosquitto_terra1_password` | MQTT password for `terra1` user |
| `mosquitto_terra2_password` | MQTT password for `terra2` user |

All variables loaded from SOPS in the playbook via the existing `community.sops.load_vars` pattern.

## Cloudflare Tunnel Update

**Not implemented in M6.** `50_apps_infra.yml` does not modify the Cloudflare Tunnel ingress configuration. Mosquitto (MQTT) is LAN-only in M6 — no public exposure via Cloudflare Tunnel.

The Cloudflare Tunnel ingress is managed exclusively by `infra/playbooks/40_platform.yml`. Exposing `mqtt.furchert.ch` publicly (including WSS + auth) is deferred as a post-M6 bundle:

- WSS endpoint (`mqtt.furchert.ch`) via Cloudflare Tunnel (Mosquitto port 9001)
- MQTT password authentication (`password_file`)
- DNS CNAME: `mqtt` → `<tunnel-id>.cfargotunnel.com` (Proxy: enabled)

This bundle requires changes to `40_platform.yml` (ingress PUT) and Mosquitto config — to be designed separately.

## Validation

```bash
# PostgreSQL — shows authdb, devicedb, datadb
kubectl port-forward -n apps svc/postgres 5432:5432
psql -h localhost -U <postgres_user> -c "\l"

# InfluxDB — returns {"status":"pass"}
kubectl port-forward -n apps svc/influxdb 8086:8086
curl http://localhost:8086/health

# Mosquitto internal
kubectl port-forward -n apps svc/mosquitto 1883:1883
mosquitto_pub -h localhost -u backend -P <password> -t test/ping -m hello

# Mosquitto external (after DNS propagation)
mosquitto_pub -h mqtt.furchert.ch -p 443 --capath /etc/ssl/certs \
  -u backend -P <password> -t test/ping -m hello --websockets
```

## Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| PostgreSQL init script runs only on first start | Low | Medium | Longhorn PVC is new — no existing data. Document in OPERATIONS.md. |
| InfluxDB re-init if PVC lost | Low | High | Restic backup covers `/var/lib/influxdb2`. |
| Mosquitto passwd file lost on pod restart | None | High | Init container regenerates it from secrets on every pod start — by design. |
| CF Tunnel ingress PUT overwrites existing rules | Medium | High | Playbook reads current config first and merges — same pattern as SSH tunnel. |
| ESP32 WebSocket TLS handshake failure | Low | Medium | Cloudflare handles TLS termination; device connects to port 443 WSS. |
